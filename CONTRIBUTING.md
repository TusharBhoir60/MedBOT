# Contributing to AarogyaAgent v2

First off, thank you for considering contributing to AarogyaAgent! It's people like you that make AarogyaAgent such a great open-source project.

## Code of Conduct
By participating in this project, you are expected to uphold our [Code of Conduct](./CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report. Following these guidelines helps maintainers understand your report, reproduce the behavior, and find related reports.
- Use the provided GitHub Issue Template.
- Explain how to reproduce the bug clearly.
- Provide logs or screenshots if possible.

### Suggesting Enhancements
- Use the provided GitHub Issue Template.
- Clearly describe the use case and the benefit of the feature.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`pytest` and `npx playwright test`).
5. Issue that pull request!

## Testing Requirements
All pull requests must pass the CI pipeline. The backend requires a 100% Pytest pass rate. Refer to the `DEVELOPER_GUIDE.md` for local testing instructions.
